# Golosuy

Веб-приложение для отметки бюллетеня на фотографии. Все данные обрабатываются **только на устройстве пользователя** — фото, EXIF и результаты CV не отправляются на сервер.

Интерфейс: **#МойГолос принадлежит мне** — mobile-first PWA на русском языке.

## Как пользоваться

1. **Загрузите фото** пустого бюллетеня — с камеры (в приложении или системной) или из галереи.
2. **Поставьте отметку** — нажмите на нужное поле на фото. Отметку можно перетащить, изменить размер или удалить. Поддерживается pinch-to-zoom.
3. **Скачайте или отправьте** результат — JPEG с наложенной галочкой или крестиком. Исходные EXIF-метаданные сохраняются, если они были в файле.

## Возможности

- Захват с камеры (`getUserMedia`, системная камера через `<input capture>`) или выбор из галереи
- Ручная расстановка отметки на бюллетене (перемещение, масштаб, удаление)
- Наложение галочки или крестика с естественным стилем (цвет чернил, наклон, «дрожание» линии)
- Скачивание и отправка через Web Share API
- Сохранение исходного EXIF-блока при экспорте в JPEG
- PWA: установка на домашний экран, офлайн-кэш статики

## В разработке

- Автоматическое обнаружение квадратиков на бюллетене (CV-модуль есть, но пока не подключён к UI)
- Docker-образ для self-hosting

## Требования

- Node.js 20+
- HTTPS для доступа к камере (на телефоне в локальной сети — обязательно)

## Разработка

Dev-сервер поднимается по **HTTPS** с самоподписным сертификатом (генерируется в `certs/` при `npm run dev`).

```bash
npm install
npm run dev
```

На компьютере: [https://localhost:5173](https://localhost:5173)

### Тест с телефона

1. Телефон и компьютер — в одной Wi‑Fi сети.
2. Запустите `npm run dev` и найдите в выводе Vite строку `Network: https://192.168.x.x:5173/`.
3. Откройте этот адрес на телефоне.
4. Примите предупреждение о самоподписном сертификате (в Safari: «Продолжить», в Chrome: «Дополнительно» → «Перейти на сайт»).

Если IP компьютера изменился, пересоздайте сертификат:

```bash
npm run dev:cert
```

## EXIF

При экспорте в итоговый JPEG вставляется **оригинальный EXIF-блок** из исходного файла (без изменения полей Make, Model, DateTimeOriginal и др.).

| Формат | Сохранение EXIF |
| ------ | --------------- |
| JPEG   | Полное (сырой APP1-сегмент) |
| PNG (eXIf) | Полное, если чанк eXIf присутствует |
| HEIC (iPhone) | Ограничено: метаданные часто недоступны как сырой блок в браузере; снимок с камеры приложения — без EXIF |
| Снимок с камеры в приложении | Без EXIF (canvas не пишет метаданные) |

## Развёртывание

Статическое приложение — достаточно отдать папку `dist/`:

```bash
npm run build
```

Подходит Cloudflare Pages, GitHub Pages, nginx. Для production **обязателен HTTPS** — иначе камера на мобильных не заработает.

### Cloudflare Pages

```bash
npm run build
npx wrangler pages deploy dist
```

Или подключите репозиторий через GitHub-интеграцию: build command — `npm run build`, output directory — `dist`.
